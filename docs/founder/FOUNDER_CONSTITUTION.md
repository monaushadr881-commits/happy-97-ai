# FOUNDER CONSTITUTION — R113

**Status:** PERMANENT LAW. Supersedes any conflicting doc except R91 Vision Lock and R111 Architecture Lock, which it incorporates by reference.

## Article I — Non-Negotiables

1. Never remove Founder-approved features.
2. Never duplicate architecture.
3. Always extend existing systems.
4. One Brain. One Memory. One Digital Human. One Workspace. One Authentication System. One Registry System (three registries only).
5. No breaking changes. Backward compatibility required.
6. Every increment must pass typecheck and all tests.

## Article II — Registries

- **Founder Registry** (`docs/founder/FOUNDER_REGISTRY.md`) — 502 modules, business architecture, never auto-generated, Founder-only modifications.
- **Technical Registry** (`docs/technical/TECHNICAL_REGISTRY.md`) — auto-scanned from source; regeneratable.
- **Feature Registry** (`docs/features/FEATURE_REGISTRY.md`) — bridges the two; every feature mapped to Founder Module + Technical Module + DB + API + Permissions + Credits + Subscription + AI + Memory + DH + Builder + Owner + Status.

## Article III — Extension Order (R114–R130)

Auth → Brain → Memory → Digital Human → Workspace → Files → Search → Builders → CRM → ERP → HRMS → Marketplace → Creator → Communication → Revenue → Enterprise Control → Founder Dashboard.

## Article IV — Quality Gates

Every PR: typecheck, tests, build verification, architecture verification (R111 §1–§7). No duplicate code, no dead code, no regressions.

## Article V — Founder Authority

- Only the Founder may add, remove, rename, or re-scope any Founder Module.
- Any deviation from this Constitution requires an explicit, written Founder Directive that supersedes it by name.
